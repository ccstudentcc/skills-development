# 中文科技论文写作规则手册

*基于《Science Research Writing》核心方法论的本土化提炼*

---

## 使用建议

> **本手册定位**：写作**方法论与结构设计**指南，核心原则（逆向工程、叙事包裹、句子功能分析）适用于中文学术写作。

> **使用方式**：
> 1. 用本手册掌握**写作思维、章节骨架、逻辑组织方法**
> 2. 写作时**务必同时参照**目标期刊的《投稿须知》或学校的《学位论文撰写规范》
> 3. 中文特有的格式规范（标点、数字用法、参考文献著录等）以官方标准（GB/T 7714、GB/T 15835等）为准

> **中英文差异说明**：本手册已针对中文写作习惯做了适配（如时态表达转化、主语归属处理等），但在具体格式层面不可替代中文格式模板。


## 〇、核心心法

### 0.1 三条根本原则

| 原则 | 英文原文 | 中文理解 |
|:---|:---|:---|
| 第一原则 | *The aim is not simply to make it possible for the reader to understand; the aim is to make it impossible for the reader NOT to understand.* | 不只要让读者“能读懂”，更要让读者“不可能读不懂” |
| 第二原则 | *Data and information alone have no intrinsic or obvious function for the reader without a narrative.* | 数据和信息本身不携带功能，必须用叙事将其包裹 |
| 第三原则 | *Good organisation and good writing can compensate for language errors, but error-free language does not compensate for poor organisation or poor writing.* | 好结构+好写作可以弥补语言瑕疵；但语言完美救不了结构混乱 |

### 0.2 逆向工程法（核心工作方法）

动笔前，必须完成以下步骤：

1. **选范本**：找3-5篇目标期刊近3-5年发表的、与你同领域的优秀论文
2. **析功能**：逐句分析每句话的**功能**，而非内容（是“确立重要性”？是“指出现有方法缺陷”？是“引出本文工作”？）
3. **建模型**：从句子功能中归纳出该章节的结构骨架
4. **采语汇**：从范本中提取实现各功能的常用词汇和句式
5. **套模型**：将自己的内容填入骨架
6. **持续更新**：随领域发展更新范本和模型

### 0.3 中文科技写作基本规范

| 规范项 | 说明 |
|:---|:---|
| 主语视角 | 推荐使用“本文”“本研究”；近年“我们”也逐渐被接受，需保持全文一致 |
| 术语一致性 | 同一概念全文用同一词汇；禁止为“避免重复”而替换近义词 |
| 缩写规范 | 首次出现写“中文全称（英文全称，缩写）”；之后只用缩写 |
| 诚实性 | 对结果、假设、局限和适用边界保持诚实；不夸大，不掩盖缺点 |
| 图表引用 | 正文中必须引用每个图表，并概括其主要结论（中文常用“如图X所示”“见表X”） |
| 章节结构 | 中文论文常见`1` `1.1` `1.1.1`三级标题；公式编号常用`(1)` `(2)`或按章节编号 |
| 数字用法 | 遵循GB/T 15835《出版物上数字用法》标准 |
| 参考文献 | 遵循GB/T 7714标准或目标期刊指定格式 |


## 一、各章节结构模型

### 1.1 引言模型（倒漏斗结构）

#### 四块骨架

| 模块 | 功能 | 中文常用句式 |
|:---|:---|:---|
| 模块1 | 确立主题重要性 + 提供背景事实 + 呈现当前研究焦点 | “近年来，……已成为……领域的研究热点。” / “……在……中发挥着关键作用。” |
| 模块2 | 呈现前人/当前研究贡献（绘制研究“地图”） | “针对该问题，已有研究主要从以下几方面展开：……” |
| 模块3 | 定位研究空白 / 描述现存问题 / 提出假设 | “然而，关于……的研究仍相对匮乏。” / “现有方法在……方面仍存在明显不足。” |
| 模块4 | 描述本文工作（目标/方法/主要发现） | “鉴于此，本文提出一种……方法，旨在……” / “本文的主要贡献包括：……” |

#### 模块2（前人研究地图）的三种组织方式

| 组织方式 | 适用场景 | 中文表达示例 |
|:---|:---|:---|
| 从一般到具体 | 最常用 | “最早由……开展了……研究。随后，……将其扩展到……。近年来，……进一步考虑了……。” |
| 按方法/理论分组 | 避免“然而……然而……”的反复转折 | “目前解决该问题的途径主要有三类：第一类基于……；第二类采用……；第三类则……” |
| 时间顺序 | 领域发展与外部事件相关 | “自2015年……政策出台以来，相关研究经历了三个阶段：……” |

#### 模块3（指出空白/问题）的常用表达

| 强度 | 中文表达 |
|:---|:---|
| 强（明确存在空白） | “目前尚无研究探讨……” / “据本文所知，尚未有工作涉及……” |
| 中（前人工作有缺陷） | “然而，上述方法在……方面仍存在局限性。” / “该方法难以适用于……” |
| 弱（存在争议或未解问题） | “……的机制尚不明确。” / “关于……的结论仍存在争议。” |

#### 模块4（引出本文工作）的常用表达

| 功能 | 中文表达 |
|:---|:---|
| 陈述目标 | “本文旨在……” / “本研究的目标是……” |
| 描述工作 | “本文提出一种……方法。” / “本文构建了……模型。” |
| 概述贡献 | “与已有工作相比，本文的主要贡献在于：……” |
| 预告结构 | “本文组织结构如下：第1节……；第2节……；……” |

### 1.2 方法模型（可复现性优先）

#### 六模块菜单（按需选用）

| 模块 | 功能 | 中文写作要点 |
|:---|:---|:---|
| 模块1 | 方法概述 / 重述目标 / 材料设备来源 | 先给“全景图”，再写细节 |
| 模块2 | 材料/方法细节 + 选择理由 + 操作严谨性标记 | 每个选择后加“以……”“旨在……”说明目的 |
| 模块3 | 图表内容描述 | “实验装置如图X所示。” |
| 模块4 | 引用前人方法 ± 比较 ± 说明修改 | 区分“完全采用”“修改采用”“显著不同”三种关系 |
| 模块5 | 用一般现在时提供背景信息 | 穿插于方法描述中的“已知事实” |
| 模块6 | 方法中的问题或局限 | 诚实说明，为未来工作埋线 |

#### 与前人方法的三种关系

| 关系 | 中文表达 |
|:---|:---|
| 完全相同 | “……参照文献[X]的方法进行。” / “……采用文献[X]所述步骤。” |
| 相似但修改 | “……在文献[X]方法的基础上进行了改进。” / “……方法基本同文献[X]，主要修改在于：……” |
| 显著不同 | “与文献[X]不同，本文采用……” / “本文未采用……，而是……” |

#### 序列语言（精确描述时序）

| 时序位置 | 中文表达 |
|:---|:---|
| 开始前 | 预先、事先、在……之前 |
| 开始时 | 首先、第一步、初始时 |
| 顺序（无精确时间） | 随后、接着、继而、然后 |
| 短暂间隔后 | 随即、立即、迅速 |
| 较长间隔后 | 之后、经……后、最终 |
| 同时/精确衔接 | 与此同时、当……时、一旦……即…… |
| 结束时 | 最后、最终 |
| 结束后 | 之后、此后 |

#### 说明选择理由的常用表达

| 功能 | 中文表达 |
|:---|:---|
| 正面说明目的 | “为……，本文采用……” / “……，以……。” |
| 说明优势 | “该方法具有……的优点。” / “选用该材料是因为其……” |
| 说明必要性 | “为保证……，需……” / “考虑到……，有必要……。” |

#### 表明操作严谨性的词汇

| 类别 | 中文表达 |
|:---|:---|
| 精度 | 精确、准确、严格 |
| 一致性 | 统一、一致、均、皆 |
| 重复性 | 重复、多次、反复 |
| 条件控制 | 恒定、稳定、严格控制 |

#### 说明问题或局限的专业表达

| 策略 | 中文表达 |
|:---|:---|
| 最小化问题 | “略有偏差”“影响可忽略”“误差在可接受范围内” |
| 归因于客观限制 | “受限于实验条件”“由于设备限制”“不可避免地” |
| 转向积极面 | “尽管存在上述不足，……仍……”“后续工作将……” |

### 1.3 结果模型（引导读者解读）

#### 四块结构

| 模块 | 功能 | 中文写作要点 |
|:---|:---|:---|
| 模块1 | 回顾目标/方法 → 总体趋势 → 引导看图 | “为验证……，本文对……进行了测试。结果如图X所示。” |
| 模块2 | 具体/关键结果 + 评价性语言 + 比较前人 + 解释 | 核心结果必须带评价（“显著”“仅”“高达”等） |
| 模块3 | 结果中的问题或局限 + 可能原因 | 诚实呈现，展示自省能力 |
| 模块4 | 结果的可能含义（为讨论做铺垫） | “以上结果表明……” / “这暗示……” |

#### 评价性语言（引导读者解读数据的方向）

| 功能 | 中文表达 |
|:---|:---|
| “增大”数值感 | 大量、显著、明显、高达、多达、大幅 |
| “减小”数值感 | 仅、只有、不到、不足、微弱的、轻微的 |
| 强调程度 | 甚至、尤其、特别、极为、极其、惊人地 |
| 表示接近 | 约、大约、接近、近乎、几乎 |
| 表示频率高 | 始终、一贯、通常、往往、频繁、经常 |
| 表示频率低 | 偶尔、间或、极少、罕见、几乎不 |
| 不做评价 | 在一定程度上、相对而言、较为 |

#### 频率描述量表（中文对应）

| 级别 | 英文 | 中文 |
|:---|:---|:---|
| 1 | always / invariably | 始终、无一例外 |
| 2 | generally / normally | 通常、一般而言 |
| 3 | frequently / often | 经常、频繁、往往 |
| 4 | more often than not | 多数情况下 |
| 5 | as often as not | 半数情况下 |
| 6 | sometimes | 有时、部分情况下 |
| 7 | occasionally | 偶尔、间或 |
| 8 | rarely / seldom | 很少、鲜有 |
| 9 | hardly ever | 几乎从不 |
| 10 | never | 从未、绝不 |

#### 引导查看图表的表达

| 功能 | 中文表达 |
|:---|:---|
| 中性引导 | “如图X所示” / “结果见图X” / “表X列出了……” |
| 强调可见性 | “从图X可以清楚地看到……” / “图X直观展示了……” |
| 引导比较 | “对比图X(a)和(b)可知……” / “图X给出了……与……的比较。” |
| 引导细节 | “图X的局部放大图显示……” / “由表X第X列数据可知……” |

### 1.4 讨论与结论模型

> **中文写作说明**：英文论文常将`Discussion`和`Conclusion`分为两节，而中文论文（尤其是学位论文）常合并为“讨论”或“结论”一节，或将“结论”作为简短收尾。以下模型提供了讨论和结论的完整组件，实际写作时**应根据目标期刊/学校模板合并或精简**。

#### 讨论的核心组件（正漏斗结构）

| 位置倾向 | 组件 | 中文表达示例 |
|:---|:---|:---|
| 开头 | 回顾本文目标或核心发现 | “本文的主要发现是……” / “本研究证实了……” |
| 开头 | 陈述成就/贡献 | “本文工作的主要贡献在于：……” |
| 开头 | “重启”读者的背景回顾 | “如引言所述，……问题一直未得到很好解决。” |
| 主体 | 解释结果（机制/原因） | “出现该现象的可能原因是……” / “这可能是由于……” |
| 主体 | 与前人研究比较 | “该结果与文献[X]的结论一致。” / “与文献[Y]报道的趋势相反，本文发现……” |
| 偏后 | 讨论局限性 | “本研究存在以下局限：……” |
| 偏后 | 未来工作建议 | “后续研究可从以下几方面展开：……” |
| 偏后/结尾 | 重申贡献/影响 | “综上所述，本文……” |
| 偏后/结尾 | 潜在应用 | “本文方法有望应用于……” |

#### 结论的常用组件（可独立成节或并入讨论末尾）

| 序号 | 组件 | 中文表达示例 |
|:---|:---|:---|
| 1 | 论文工作概述 | “本文研究了……” / “本文围绕……开展了……工作。” |
| 2 | 主要成果 | “本文成功实现了……” / “本文首次……” |
| 3 | 关键结果总结 | “实验结果表明，……显著提升/降低了……。” |
| 4 | 研究意义 | “本文深化了对……的认识。” |
| 5 | 应用前景 | “本文方法可应用于……” |
| 6 | 未来方向 | “后续研究可关注……” |

#### 与前人研究的六种关系定位

| 关系 | 中文表达 |
|:---|:---|
| 验证/支持 | “这与文献[X]的结论一致。” / “本文结果进一步印证了……” |
| 扩展/推进 | “本文在文献[X]的基础上，进一步……” / “本文将……扩展到……” |
| 挑战/矛盾 | “与文献[X]的结论不同，本文发现……” / “本文结果对……观点提出了质疑。” |
| 互补 | “本文工作与文献[X]的工作互为补充：……” |
| 提供新视角 | “本文为理解……提供了新的视角。” / “本文揭示了……的新机制。” |
| 开辟道路 | “本文工作为后续……研究奠定了基础。” |

### 1.5 摘要模型

#### 三类内容模块

| 模块 | 内容 | 中文写作要点 |
|:---|:---|:---|
| 模块1 | 主题重要性/背景 / 问题 / 本文工作概述 | 首句快速切入，避免空泛铺垫 |
| 模块2 | 方法要点 / 关键结果（带数据） / 结果含义 | 数字必须带评价（“高达”“仅为”） |
| 模块3 | 贡献定位 / 价值陈述 / 应用前景 | 明确说出“首次”“优于”“可用于” |

#### 摘要特有约束

| 约束项 | 规范 |
|:---|:---|
| 明确归属 | 需清晰区分“本文工作”与“已知事实”。中文摘要传统上较少反复使用“本文”，可根据目标期刊风格选择无主语句（“采用……方法，对……进行了研究，结果表明……”）或明确主语。 |
| 引用回避 | 摘要中一般不出现文献引用 |
| 方法精简化 | 只保留判断是否读全文所需的关键方法信息 |
| 数字加评价 | 禁止裸数字 |
| 独立可读 | 假设读者只读摘要，必须包含完整逻辑链 |

#### 摘要中表达成就/价值的常用词

| 类别 | 中文表达 |
|:---|:---|
| 新颖性 | 首次、新的、新型的、新颖的 |
| 优越性 | 优于、显著提升、大幅降低、高效、精确 |
| 可靠性 | 鲁棒的、稳定的、可靠的 |
| 实用性 | 可用于、适用于、有望应用于 |

> **注意**：使用“首次”“首个”等绝对化表述前，务必进行全面文献检索，确保有充分依据。


## 二、时态与时间意义的表达（中文视角）

### 2.1 中文时间意义的表达方式

中文无动词形态变化，时间意义通过时间副词、助词（“了”“过”“着”）及语境表达。在科技写作中，区分“本研究发现”与“普遍事实”尤为重要。

| 时间意义 | 中文表达方式 | 通信含义 |
|:---|:---|:---|
| 本研究的具体发现 | “实验结果显示……” / “本研究发现……” / “测得……值为……” | 仅报告本研究中发生的事件；不宣称普遍性 |
| 可视为普遍事实 | “……是……”“……呈……关系”“……具有……性质” | 断言具有跨研究、跨条件的普遍性（高风险，需充分证据） |
| 当前仍相关的状态 | “已有研究表明……” / “近年来，……引起了……” / “尚无……的报道” | 暗示该问题仍是当前研究前沿，本文即将回应 |

### 2.2 各章节时间意义倾向参考

| 章节 | 时间意义倾向 | 中文表达示例 |
|:---|:---|:---|
| 引言-背景事实 | 普遍事实 | “……是一种……” / “……在……中发挥着关键作用。” |
| 引言-前人研究 | 具体研究发现 | “文献[X]报道了……” / “文献[X]发现……与……有关。” |
| 引言-当前空白 | 当前仍相关 | “尚无研究系统探讨……” / “……的机制尚不明确。” |
| 引言-本文工作 | 普遍/当前 | “本文提出……” / “本文旨在……” |
| 方法-本文操作 | 本研究具体操作 | “将样品置于……℃下加热……小时。” / “采用……对……进行测量。” |
| 方法-标准程序/原理 | 普遍事实 | “该方法的原理是……” / “……的熔点为……℃。” |
| 结果-图表描述 | 普遍（图表内容） | “图X展示了……” / “由表X可知……” |
| 结果-数据获得过程 | 本研究具体操作 | “在……条件下记录到……” / “当……时，观察到……” |
| 讨论-解释结果 | 普遍或推测 | “这可能是由于……” / “一种可能的解释是……” |
| 结论-陈述成果 | 普遍/当前 | “本文实现了……” / “本文提出了……” |


## 三、模态与确定性表达

### 3.1 中文表达确定性的连续体

| 确定性 | 中文表达 | 适用场景 |
|:---|:---|:---|
| 最高（事实证明） | “结果证明……”“实验证实……”“数据表明……” | 有充分证据，结论可直接从数据得出 |
| 高（高度确信） | “由此可见……”“因此……”“这表明……” | 逻辑链条清晰，但留有极少解释空间 |
| 中（有把握但留有余地） | “结果表明……可能……”“这暗示……”“这反映出……可能……” | 数据支持，但存在其他解释可能 |
| 低（推测） | “我们推测……”“一种可能的解释是……”“这似乎与……有关” | 数据只提供间接支持，需更多验证 |
| 最低（开放性假设） | “这是否意味着……仍有待进一步研究。”“……尚不明确。” | 仅为猜想或提出问题 |

### 3.2 中文表达“可能性”的层级

| 可能性程度 | 中文表达 |
|:---|:---|
| 极可能 | 几乎肯定、极有可能、大概率 |
| 很可能 | 很可能、大概率、有望 |
| 可能 | 可能、或许、或可 |
| 不太可能 | 不太可能、可能性较低 |
| 极不可能 | 几乎不可能、可能性极低 |

### 3.3 风险降级策略（中文版）

当对结论信心不足但又要陈述时，可采用以下方式降低断言强度：

| 策略 | 示例 |
|:---|:---|
| 加频率修饰 | “在多数情况下……” / “部分样本中……” |
| 加范围修饰 | “在一定范围内……” / “在……条件下……” |
| 使用推测动词 | “似乎”“可能”“或许” |
| 使用框架短语 | “有证据表明……” / “初步结果显示……” |
| 归因于有待验证 | “该现象的机制有待进一步研究。” |


## 四、因果关系的精确表达

### 4.1 因果强度分级

| 强度 | 中文表达 | 说明 |
|:---|:---|:---|
| 直接/强因果 | 导致、引起、造成、缘于、归因于 | 明确的因果关系 |
| 部分因果 | 促进、有助于、参与、在一定程度上导致 | 多因素中的一个 |
| 因果链条 | 进而、从而、由此、继而 | 隐含过程性 |
| 初始原因 | 源于、发端于、起始于 | 追溯源头 |
| 弱/隐含关联 | 与……有关、与……相关、与……存在联系 | 不明确因果方向 |
| 统计关联 | 与……呈正相关、与……呈负相关、与……无显著相关性 | 统计关系，不一定是因果关系 |

### 4.2 因果方向必须明确

| 可能歧义的表达 | 明确表达 |
|:---|:---|
| “A和B有关” | “A与B呈正相关” / “A导致B” / “B影响A” |
| “A影响B” | “A促进B” / “A抑制B” |
| “A的变化引起B的变化” | “A增加导致B增加” / “A升高伴随B降低” |

### 4.3 区分“原因之一”与“唯一原因”

| 表达 | 含义 |
|:---|:---|
| “A是B的一个原因” | A是诸多原因之一 |
| “A是B的主要原因” | A贡献最大，但非唯一 |
| “A是B的直接原因” | A与B有直接因果链条 |
| “A是B的唯一原因” | 排除其他所有原因（极少用，除非有严格对照实验） |


## 五、“所有权”管理：区分“本文工作”与“已知事实”

### 5.1 中文的“主语缺失”歧义

中文科技写作中，最易出现的“所有权”歧义是：**当主语为无生命名词时，读者无法判断该陈述是“本文的发现”还是“已知事实”**。

| 歧义句 | 可能解读一 | 可能解读二 |
|:---|:---|:---|
| “理论建模表明，该结构具有更高的稳定性。” | 这是本文的建模结果 | 这是已有理论的一般结论 |
| “分析发现，两组间存在显著差异。” | 这是本文的分析结果 | 这是前人文献的结论 |
| “实验证实该假设成立。” | 这是本文的实验 | 这是领域共识 |

### 5.2 消除歧义的五种方法

| 方法 | 示例 |
|:---|:---|
| 1. 加“本文”“本研究” | “本文的理论建模结果表明……” / “本研究的分析发现……” |
| 2. 使用“了”标记本研究事件 | “实验证实了该假设。” / “分析发现了显著差异。” |
| 3. 使用“……结果”作主语 | “本文的实验结果证实……” / “建模结果显示……” |
| 4. 对于已知事实，加文献引用或“已有研究表明” | “已有研究表明，该结构具有更高的稳定性[3]。” |
| 5. 对于已知事实，使用“一般认为”“众所周知” | “一般认为，该现象由……引起。” |

### 5.3 “本文”与“我们”的使用建议

| 主语 | 使用建议 |
|:---|:---|
| “本文” | 传统中文科技写作推荐，陈述论文内容、方法、结果、贡献 |
| “本研究” | 同上，强调研究工作本身 |
| “我们” | 近年逐渐被接受，尤其在多人合作论文中；需保持全文一致，不频繁切换 |
| 无主语句 | 中文摘要中常见（“采用……方法，对……进行了研究”），但需确保不产生歧义 |


## 六、段落与句子结构

### 6.1 段落原则

| 原则 | 说明 |
|:---|:---|
| 单一功能 | 每个段落只完成一个明确的交际功能 |
| 首句定向 | 段落首句应提示该段的主题或走向 |
| 长度适中 | 中文科技论文段落一般150-300字；避免整页长段或连续短段 |
| 可skim | 将各段首句连起来应能构成全文逻辑脉络 |

### 6.2 段落首句的常用模式

| 模式 | 示例 |
|:---|:---|
| 主题预告式 | “本节讨论……对……的影响。” |
| 承上启下式 | “在明确……之后，接下来分析……” |
| 问题导入式 | “上述结果引出一个问题：……” |
| 转折推进式 | “与前述……不同，……表现出……” |
| 结论预告式 | “综合以上分析，可得出以下结论：……” |

### 6.3 句子连接四法（中文版）

| 方法 | 说明 | 示例 |
|:---|:---|:---|
| 重叠重复 | 将上一句关键词重复于下一句开头 | “……进入无意识状态。无意识状态可在数十秒内发生。” |
| 指代 | 使用“该方法”“上述结果”“这一现象”等 | 注意：指代对象必须明确唯一 |
| 分号连接 | 两句关系极紧密时用分号 | “实验进行了三次；三次结果均一致。” |
| 关联词引导 | 使用“然而”“因此”“此外”“换言之”等 | 必须准确反映逻辑关系 |

### 6.4 关联词精确分类

| 逻辑关系 | 关联词 | 注意事项 |
|:---|:---|:---|
| 因果 | 因此、因而、故而、由此 | 必须有真实因果关系，不可滥用 |
| 转折 | 然而、但、但是、却、不过 | 区分“对比差异”和“意外发现” |
| 递进 | 此外、而且、进而、不仅如此 | 后句必须是前句的进一步推进 |
| 并列/列举 | 首先……其次……最后；一方面……另一方面 | 保持结构对称 |
| 举例 | 例如、比如、以……为例 | 必须是前句概括的具体实例 |
| 解释/换言 | 即、换言之、也就是说 | 后句必须是前句的同义重述 |
| 总结 | 综上、总而言之、由此可见 | 用于段落或章节收束 |

### 6.5 句子长度参考

| 句子长度（中文字符） | 可读性 |
|:---|:---|
| ≤30字 | 最易读，首次阅读可理解率>90% |
| 30-50字 | 适中 |
| 50-80字 | 偏长，建议检查是否可拆分（中文科技论文中此长度常见，以清晰为首要目标） |
| >80字 | 过长，强烈建议拆分或重构 |

### 6.6 常见中文句式问题及修正

| 问题类型 | 问题句示例 | 修改建议 |
|:---|:---|:---|
| 主语过长 | “采用基于深度学习框架并结合注意力机制和多尺度特征融合的图像分割方法对医学影像进行了分析。” | 拆分或前置：“本文提出一种图像分割方法，该方法基于深度学习框架，结合注意力机制与多尺度特征融合。将该方法应用于医学影像分析，结果表明……” |
| “的”字堆积 | “本文提出的基于改进的深度神经网络的医学图像的分割的方法的性能的评估” | 减少“的”字：“本文提出一种改进的深度神经网络医学图像分割方法，并对其性能进行评估。” |
| 动宾相隔太远 | “本文对在多种光照条件下采集的不同分辨率的自然图像数据集进行了测试。” | 将状语提前或拆分：“本文在多种光照条件下采集了不同分辨率的自然图像数据集，并对其进行了测试。” |
| 多重否定 | “不排除该现象并非由温度变化引起的可能性。” | 改为肯定表达：“该现象可能由温度变化引起。” |


## 七、图表与数据的呈现

### 7.1 图表引导的原则

| 原则 | 说明 | 正例 | 反例 |
|:---|:---|:---|:---|
| 正文引用 | 正文中必须引用每个图表 | “实验结果如图X所示。” | 图表出现但正文从未提及 |
| 引用中概括趋势 | 提及图表时给出主要结论 | “图X显示，A组显著高于B组。” | “图X给出了A组和B组的比较。” |
| 不做细节复述 | 正文不重复图表中每个数据点 | 概括趋势和关键数据 | 罗列图表中所有数值 |
| 数字带评价 | 关键数值必须附加评价 | “A组高达47%，而B组仅为23%。” | “A组为47%，B组为23%。” |

> **中文写作注意**：中文期刊的图表位置由编辑最终确定，作者只需确保正文中有明确引用（“如图X所示”“见表X”），不必追求“先提及后出现”的精确控制。

### 7.2 图表标题规范

| 类型 | 规范 | 示例 |
|:---|:---|:---|
| 图标题 | 置于图下方；以“图X”开头；能独立理解 | “图3 不同温度下材料的应力-应变曲线” |
| 表标题 | 置于表上方；以“表X”开头；能独立理解 | “表2 各实验组的样本量及基线特征” |

### 7.3 数据呈现的精度规范

| 数据类型 | 小数位数 | 示例 |
|:---|:---|:---|
| 百分比（样本量足够） | 一般1位小数 | 23.5% |
| 百分比（样本量小） | 整数或分数 | 3/10（而非30%） |
| p值 | 2-3位小数 | p=0.032 |
| 测量值 | 与测量精度匹配 | 根据仪器精度决定 |
| 统计量（t/F/χ²） | 2位小数 | t=2.34 |


## 八、引用规范

### 8.1 引用位置与信息归属

| 引用位置 | 含义 | 示例 |
|:---|:---|:---|
| 紧跟所支持的信息 | 仅该部分信息来自该文献 | “……已有研究发现A与B呈正相关[3]，但关于C的作用尚不明确。” |
| 置于句末 | 整句信息均来自该文献 | “深度学习方法在医学图像分割中取得了优异性能[4-6]。” |
| 多个引用堆叠 | 需确认是否每个引用都支持整句 | 若各文献贡献不同，应分开引用 |

### 8.2 引用的功能性分类

| 引用功能 | 中文表达示例 |
|:---|:---|
| 提供背景 | “……的研究可参见文献[X]的综述。” |
| 支持观点 | “这与文献[X]的结论一致。” |
| 对比/反驳 | “与文献[X]的结论不同，本文发现……” |
| 方法来源 | “本文采用文献[X]提出的方法。” |
| 方法比较 | “与文献[X]的方法相比，本文方法在……方面有所改进。” |
| 数据比较 | “本文结果与文献[X]报道的……基本吻合。” |

### 8.3 避免“引用堆砌”的写作模式

| 不佳模式 | 改进模式 |
|:---|:---|
| “……已得到广泛研究[1-15]。” | “……已得到广泛研究，主要包括A方向[1-5]、B方向[6-10]以及C方向[11-15]。” |
| “许多方法被提出[1,3,5,7,9]。” | “针对该问题，研究者先后提出了基于A的方法[1,3]、基于B的方法[5]以及基于C的方法[7,9]。” |


## 九、术语与词汇规范

### 9.1 术语一致性原则

| 原则 | 说明 |
|:---|:---|
| 一概念一词 | 同一概念全文使用同一术语；不因“避免重复”而替换 |
| 首次定义后沿用 | 缩写或简称首次定义后，之后统一使用 |
| 禁止同义词替换 | “方法”不变为“途径”→“手段”→“框架”；“模型”不变为“模式”→“架构” |
| 新术语明确定义 | 本文提出或引入的新术语，必须在首次出现时明确定义 |
| 专业名词以审定为准 | 中文科技名词应以全国科学技术名词审定委员会公布的为准 |

### 9.2 缩写使用规范

| 步骤 | 做法 | 示例 |
|:---|:---|:---|
| 首次出现 | 写“中文全称（英文全称，缩写）” | “卷积神经网络（Convolutional Neural Network, CNN）” |
| 后续出现 | 仅用缩写 | “CNN” |
| 摘要中 | 同正文规范 | 首次出现写全称 |
| 标题中 | 尽量避免缩写，除非领域内绝对通用 | “基于CNN的图像分割”（若CNN极通用） |

### 9.3 中文科技写作易混词辨析

| 词对 | 区别 |
|:---|:---|
| 实验 vs 试验 | “实验”偏科学验证；“试验”偏尝试性探索 |
| 数据 vs 数值 | “数据”是集合概念；“数值”是具体的数 |
| 参数 vs 变量 | “参数”在特定问题中固定；“变量”可变化 |
| 方法 vs 算法 | “方法”泛指；“算法”特指计算步骤 |
| 模拟 vs 仿真 | 基本同义，“仿真”偏工程应用 |
| 显著 vs 明显 | “显著”有统计含义；“明显”泛指程度大 |
| 相关 vs 有关 | “相关”有统计含义；“有关”泛指存在联系 |
| 证明 vs 表明 | “证明”逻辑严格；“表明”有推断成分 |
| 包括 vs 包含 | “包括”可指部分列举；“包含”常指全部要素 |
| 即 vs 就是 | “即”用于书面解释；“就是”偏口语 |

### 9.4 应避免的口语化表达

| 口语化 | 书面化 |
|:---|:---|
| 搞/做（实验） | 进行、开展、实施 |
| 看（出） | 观察、发现、可知 |
| 很多 | 大量、众多、诸多 |
| 有点 | 略有、稍显、在一定程度上 |
| 越来越 | 日益、逐渐、持续 |
| 差不多 | 大致、约、近似 |
| 结果挺好 | 结果良好、取得了较好效果 |
| 不太行 | 效果欠佳、性能有待提升 |


## 十、检查清单

### 10.1 结构检查

| 检查项 | 自问 |
|:---|:---|
| 逆向工程 | 是否参照了目标期刊/学校的近期范本？ |
| 各章节模型完整 | 引言是否四块俱全？讨论是否与引言对称？ |
| 段落功能单一 | 每段是否只完成一个交际任务？ |
| 段落首句定向 | 连读各段首句能否形成逻辑脉络？ |

### 10.2 读者导向检查

| 检查项 | 自问 |
|:---|:---|
| 背景充分 | 跨领域读者能否读懂？是否假设了过多先验知识？ |
| 信息功能明确 | 每句话的功能是否清楚？读者是否知道“为什么这句话在这里”？ |

| “so what”消解 | 每段结束时，读者是否会有“所以呢”的疑问？ |
| 所有权明确 | 读者能否清楚区分“本文工作”和“已知事实”？ |

### 10.3 句子检查

| 检查项 | 自问 |
|:---|:---|
| 句子连接 | 句与句之间是否有明确连接（重复/指代/关联词）？ |
| 长度与清晰度 | 是否有超过80字仍不清晰的句子？是否需要拆分？ |
| 主语清晰 | 每句主语是否明确？主语视角全文是否一致？ |
| 动宾不分离 | 动词和宾语是否被过长状语隔开？ |
| 关联词准确 | “因此”前后是否真有因果关系？“例如”是否真是实例？ |

### 10.4 术语与表达检查

| 检查项 | 自问 |
|:---|:---|
| 术语一致 | 同一概念是否通篇用同一词汇？ |
| 缩写规范 | 缩写首次出现是否给了全称？ |
| 形容词节制 | 是否有空泛的“很好”“极大提升”？是否有数据支撑？ |
| 口语化 | 是否有“搞”“做”“挺”“有点”等口语词？ |
| 诚实性 | 结论是否与数据匹配？局限是否诚实说明？ |

### 10.5 图表检查

| 检查项 | 自问 |
|:---|:---|
| 图标题完整 | 图标题是否能独立理解？ |
| 表标题完整 | 表标题是否能独立理解？ |
| 正文引用 | 正文中是否引用了每个图表并概括趋势？ |
| 精度恰当 | 小数位数是否与测量精度/样本量匹配？ |
| 评价伴随 | 关键数值是否附加了评价语言？ |

### 10.6 引用检查

| 检查项 | 自问 |
|:---|:---|
| 引用位置准确 | 引用是否紧邻其所支持的信息？ |

## 十一、全书核心金句（中文译本）

| 序号 | 英文原文 | 中文译文 |
|:---|:---|:---|
| 1 | *The aim is not simply to make it possible for the reader to understand; the aim is to make it impossible for the reader NOT to understand.* | 目标不是让读者“能读懂”，而是让读者“不可能读不懂”。 |
| 2 | *Data and information alone have no intrinsic or obvious function for the reader without a narrative.* | 没有叙事包裹的数据和信息，对读者而言没有内在的、显见的功能。 |
| 3 | *Knowing WHAT to write is not the same as knowing HOW to write it.* | 知道“写什么”不等于知道“怎么写”。 |
| 4 | *The fact that you and your colleagues understand what you have written makes it harder for you to see potential ambiguities.* | 正因为你和同事都理解你写的东西，你反而更难发现潜在的歧义。 |
| 5 | *Good organisation and good writing can compensate for language errors, but error-free language does not compensate for poor organisation or poor writing.* | 好结构与好写作可以弥补语言错误，但毫无语病的语言救不了糟糕的结构和写作。 |
| 6 | *Academic English is no-one's first language.* | 学术英语不是任何人的母语。（同理：学术中文也不是任何人的母语。） |
| 7 | *Science does not exist until it is published.* | 发表之前，科学并不存在。 |

## 附录：常用功能句式速查

### A. 确立重要性
- “近年来，……已成为……领域的研究热点。”
- “……在……中发挥着至关重要的作用。”
- “随着……的发展，……问题日益突出。”

### B. 指出研究空白
- “然而，关于……的研究仍相对匮乏。”
- “目前尚无研究系统探讨……与……的关系。”
- “现有方法在……方面仍存在明显不足。”

### C. 引出本文工作
- “鉴于此，本文提出一种……方法，旨在……”
- “本文围绕……问题，开展了以下工作：……”
- “与已有工作不同，本文……”

### D. 描述方法
- “为验证……，本文设计了……实验。”
- “……的制备流程如下：……”
- “采用……（型号，厂商，产地）对……进行表征。”

### E. 呈现结果
- “实验结果如图X所示。可以看出，……”
- “当……时，……显著升高/降低（p<0.05）。”
- “值得注意的是，……组的值仅为对照组的……%。”

### F. 解释/讨论
- “出现该现象的可能原因是……”
- “这一结果与文献[X]的结论一致/不一致，可能的原因在于……”
- “上述结果表明……可能……。”

### G. 陈述贡献
- “本文的主要贡献可归纳为以下几点：……”
- “与已有方法相比，本文方法在……方面提升了……%。”
- “本文工作为……提供了新的思路/方法/依据。”

### H. 说明局限
- “本研究存在以下局限：第一，……；第二，……。”
- “受限于……，本文未能……”
- “……问题仍有待进一步研究。”

### I. 展望未来
- “后续研究可从以下几方面展开：……”
- “本文方法有望推广至……”
- “下一步工作将聚焦于……”
