# Contributing

## Workflow

1. Define scope in `SPEC.md`, `IMPLEMENTATION_PLAN.md`, and `TASK_STATUS.md` for non-trivial work.
2. Keep new skills self-contained under `skills/<skill-name>/`.
3. Prefer small, reversible edits and validate before committing.

## Quality Bar

- Keep `SKILL.md` concise and move heavy detail into `references/`.
- Ensure every bundled file is referenced from the skill.
- Avoid placeholder content, unsupported claims, and unsafe instructions.
- Run the available skill validator before publishing.
